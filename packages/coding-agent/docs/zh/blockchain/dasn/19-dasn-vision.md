# DASN: 去中心化 Agent 服务网络

> Decentralized Agent Service Network - 基于 pi-mono 的 Agent 经济网络架构愿景

---

## 核心洞察

**pi-worker + 区块链 ≠ 简单的任务外包**

**pi-worker + 区块链 = 去中心化 Agent 服务网络 (DASN)**

这是一个全新的架构范式：

```
传统模式                    DASN 模式
─────────────────────────────────────────────────
中心化 AI 服务     →       去中心化 Agent 网络
(OpenAI API)               (pi-worker 网络)
                           
单一供应商         →       多 Worker 竞争
(垄断定价)                 (市场定价)
                           
黑盒执行           →       可验证执行
(信任服务商)               (链上证明)
                           
固定能力           →       专业化分工
(通用模型)                 (专家 Worker)
```

---

## 什么是 DASN

DASN 是一个三层架构的去中心化网络：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: 应用层 (Applications)                              │
│  ├── 代码生成 Agent                                          │
│  ├── 代码审查 Agent                                          │
│  ├── 文档生成 Agent                                          │
│  └── 测试 Agent                                              │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 服务层 (Agent Services)                            │
│  ├── pi-worker Runtime (执行引擎)                            │
│  ├── Skill Registry (能力注册)                               │
│  ├── Reputation System (声誉系统)                            │
│  └── Quality Assurance (质量保证)                            │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 协调层 (Coordination)                              │
│  ├── Task Marketplace (任务市场)                             │
│  ├── Payment Settlement (支付结算)                           │
│  ├── Dispute Resolution (争议解决)                           │
│  └── Staking & Slashing (质押惩罚)                           │
│  ─────────────────────────────────                          │
│  区块链: Solana / Sui / 其他                                 │
└─────────────────────────────────────────────────────────────┘
```

### 参与者角色

| 角色 | 功能 | 激励 |
|------|------|------|
| **Client** | 发布任务需求 | 获得高质量 Agent 服务 |
| **Worker** | 运行 pi-worker 执行任务 | 获得代币奖励 |
| **Reviewer** | 仲裁争议、验证质量 | 获得仲裁费用 |
| **Developer** | 开发 Skill/扩展 | 获得使用费分成 |
| **Staker** | 质押代币支持网络 | 获得网络收益分成 |

## 与传统去中心化计算的区别

### 不是简单的"算力市场"

| 维度 | 传统去中心化计算 | DASN |
|------|-----------------|------|
| **资源类型** | 算力、存储、带宽 | Agent 能力、知识、推理 |
| **任务单位** | 确定性计算 | 非确定性 LLM 推理 |
| **验证方式** | 可复现计算 | 结果满意度 + 经济博弈 |
| **专业化** | 同质化资源 | 异质化能力 |
| **定价模型** | 按资源消耗 | 按价值创造 |

### 为什么是 Agent 网络而非算力网络

1. **AI 服务的本质是推理，不是计算**
   - 同样的输入可能产生不同输出
   - 质量取决于模型、Prompt、上下文
   - 无法简单验证计算过程

2. **Agent 需要持续状态**
   - 多轮对话
   - 工具调用链
   - 上下文维护

3. **质量是多维的**
   - 正确性
   - 效率
   - 代码风格
   - 可维护性

```
Client 支付
    │
    ├──▶ Worker (80-90%) - 执行奖励
```

| **Reviewer 质押** | 仲裁公正性 | 仲裁费 | 错误判罚被罚 |
| **Client 质押** | 防止恶意任务 | 无 | 任务无效被罚 |

### DASN 扩展模块

```typescript
// extensions/dasn-connector.ts
export default function (pi: ExtensionAPI) {
  // 1. 链上任务监听
  pi.on("session_start", async (_event, ctx) => {
    const chainClient = new SuiChainClient(config);
    
    // 订阅任务事件
    chainClient.subscribeEvent('TaskCreated', async (task) => {
      if (canHandle(task, ctx)) {
        // 自动认领
        await claimTask(task.id);
        
        // 执行
        const result = await executeTask(task, ctx);
        
        // 提交结果
        await submitResult(task.id, result);
      }
    });
  });
  
  // 2. 声誉管理
  pi.on("task_complete", async (event, ctx) => {
    // 更新本地声誉
    updateReputation(event.taskId, event.success);
    
    // 同步到链上
    await syncReputationToChain();
  });
}
```

## 与现有概念的对比

### vs MCP (Model Context Protocol)

| 方面 | MCP | DASN |
|------|-----|------|
| **定位** | 协议标准 | 经济网络 |
| **关系** | 客户端-服务器 | 去中心化市场 |
| **付费** | 订阅/按量 | 竞价/拍卖 |
| **信任** | 信任服务商 | 经济博弈 |

**关系**: MCP 可以作为 DASN 中 Worker 和 Client 之间的通信协议

### vs 传统外包平台 (Upwork/Fiverr)

| 方面 | 传统平台 | DASN |
|------|----------|------|
| **中介** | 中心化平台 | 智能合约 |
| **费率** | 20% | 5-10% |
| **全球接入** | 受限 | 无许可 |
| **支付** | 法币，延迟 | 加密货币，即时 |
| **争议** | 平台裁决 | 社区仲裁 |

### vs 去中心化 AI (Bittensor)

| 方面 | Bittensor | DASN |
|------|-----------|------|
| **焦点** | 模型训练 | Agent 服务 |
| **验证** | 梯度验证 | 结果满意度 |
| **用户** | 模型使用者 | 终端用户 |
| **技术栈** | 专用链 | 通用链 + pi-mono |

## 技术挑战

### 1. 质量验证

**问题**: LLM 输出难以客观验证

**解决方案**:
- 多重验证: 创建者验收 + Reviewer 抽检
- 声誉博弈: Worker 长期声誉积累
- 经济惩罚: 质押 slash 机制

### 2. 隐私保护

**问题**: 代码/数据上传链上公共存储

**解决方案**:
- TEE (可信执行环境)
- 零知识证明 (zk-SNARKs)
- 私有 IPFS 网络

### 3. 延迟问题

**问题**: 区块链交互增加延迟

**解决方案**:
- 乐观执行 (先执行后验证)
- Layer 2 扩展
- 状态通道

### 4. 标准化

**问题**: Agent 能力难以标准化描述

**解决方案**:
- Skill 标准 (类似 MCP)
- 能力图谱
- 基准测试集

## 总结

DASN 不是简单的"用区块链做 AI"，而是：

> **构建一个去中心化的、开放的、竞争性的 Agent 服务市场，让任何人可以提供服务，任何人可以使用服务，通过经济激励和博弈机制保证质量。**

这是一个全新的计算范式：
- **从云计算到边缘计算**
- **从中心化服务到去中心化网络**
- **从 API 调用到经济协作**

pi-mono 是这个网络的基石，提供了最核心的 Agent Runtime 能力。
