### `zig/src/cli/`

12 files, ~3370 LOC. Largest: `args.zig` (879). Likely no major refactoring
needed; defer until after `ai/` cleanup.

### `zig/src/coding_agent/`

~80 files, ~75000+ LOC. Subdirectories:

- `interactive_mode/` — TUI-driven REPL (largest single subdomain)
- `modes/` — non-interactive modes (RPC, MCP stdio, print)
- `sessions/` — session lifecycle, search, persistence (jsonl, html export)
- `extensions/` — extension protocol, runtime, registry, native + wasm hosts
- `auth/` — auth flows
- `config/` — config + migrations
- `resources/` — system prompt, file/image resources
- `tools/` — built-in tools (read/write/edit/bash/grep/find/ls/truncate/...)
- `packages/` — package manager
- `tests/` — integration test harnesses

This domain is **out of scope** for the current review pass. Address only after
`ai/` and `agent/` are stable.

### `zig/src/tui/`

22 files, ~9520 LOC. Pure render layer. Largest: `rendering.zig`-style files
inside `coding_agent/interactive_mode/` actually own the rendering hot path,
not these reusable components.

These groups have visible parallel structure across providers. They are the
**top targets** for shared-layer extraction (see
[`02_provider_duplication.md`](./02_provider_duplication.md)).

   - `openai_responses.zig`
   - `openai_codex_responses.zig`
   - `azure_openai_responses.zig`

2. **`finalizeOutputFromPartials` family** — appears in:
   - `openai_responses.zig`
   - `openai_codex_responses.zig`
   - `azure_openai_responses.zig`
   - `bedrock.zig`
   - `kimi.zig`

3. **`parseSseStreamLines` outer loop** — appears in nearly every provider.

4. **`mapStopReason`** — appears in `anthropic.zig`, `bedrock.zig`,
   `openai_responses.zig`, `openai_codex_responses.zig`,
   `azure_openai_responses.zig`, `mistral.zig`, `openai_chat_sse.zig`.

5. **Streaming JSON args repair / `parseStreamingJsonToValue`** — appears in
   `openai_chat_sse.zig`, `kimi.zig`, anthropic-style providers.

6. **`emitRuntimeFailure`** — appears in `openai_responses.zig`,
   `openai_codex_responses.zig`, `azure_openai_responses.zig`, `kimi.zig`.
