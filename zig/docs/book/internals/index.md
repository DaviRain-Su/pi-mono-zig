# 内部模块卷宗

这一组文档不是给读者读的"教材"，而是 **pi-mono-zig 自己的内部图纸**。每一份卷宗精读一个模块，输出四件东西：

1. **职责**：这个模块对外承诺什么、不承诺什么。
2. **形状**：内部数据结构、控制流、依赖关系——配架构图。
3. **C ABI 友好度**：未来要把核心暴露成 C 接口供其他语言绑定（Go / Rust / Python / Swift 等），这个模块当前哪些地方"过不去 ABI"，需要怎么改。
4. **设计气味**：诚实记录现在不够好的地方，作为后续重构的入口。
